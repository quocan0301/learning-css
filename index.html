1. Lịch sử về CSS

a. CSS là gì?

CSS là chữ viết tắt của Cascading Style Sheets, có nghĩa là các tập tin định kiểu theo tầng.

CSS có các code để tạo kiểu cho nội dung trang web, cho phép tạo ra các trang web đẹp mắt.

Tương tự như HTML thì CSS không phải là ngôn ngữ lập trình, mà là style sheet language (ngôn ngữ định dạng trang). CSS sẽ tạo kiểu cho các phần tử của HTML một cách có chọn lọc.

Khái niệm của nó được bắt nguồn bởi Håkon Wium Lie vào năm 1994. Vào tháng 12 năm 1996, CSS đã được W3C tạo ra một đặc tả và ngày nay cho phép các nhà phát triển web thay đổi bố cục và diện mạo của các trang web.

CSS lần đầu tiên được đề xuất bởi Håkon Wium Lie vào ngày 10 tháng 10 năm 1994.

b. Lịch sử phiên bản

Có 3 phiên bản chính:

  • CSS W3C phiên bản đầu tiên (CSS1) được xuất bản vào ngày 17 tháng 12 năm 1996.

  • CSS cấp 2 được W3C phát triển vào tháng 5 năm 1998. Nó là phiên bản nâng cấp từ CSS1, ngoài ra CSS 2 còn bao gồm một số khả năng mới

    Css 2.1 được nâng cấp, sửa lỗi loại bỏ những tính năng kém hoặc không tương thích đầy đủ cho người dùng, sau đó được đề xuất vào ngày 12 tháng 4 năm 2011, và chỉ 2 tháng sau nó đã được đưa đến người dùng một lần nữa.

  • CSS 3 được chia thành nhiều tài liệu riêng biệt gọi là “module”. Mỗi module thêm các khả năng mới hoặc mở rộng các tính năng được xác định trong CSS 2, duy trì khả năng tương thích ngược.

    Bản nháp CSS 3 sớm nhất đã được xuất bản vào tháng 6 năm 1999. CSS3 là phiên bản CSS (Cascading Style Sheets) thay thế CSS2 . Nó giới thiệu các bộ chọn và thuộc tính mới cho phép linh hoạt hơn với bố cục và trình bày trang.

5. CSS Selectors

  1. Simple CSS Selectors

    a. Element selector

    Element selector: chọn các phần tử HTML dựa trên tên phần tử.

    p {
  text-align: center;
  color: red;
}

    b. Id selector

    CSS id Selector: chọn các phần tử HTML dựa trên id của các phần tử (#).

    #para1 {
  text-align: center;
  color: red;
}

    c. Class selector

    CSS class Selector: chọn các phần tử HTML dựa trên tên class của các phần tử (.).

    .center {
  text-align: center;
  color: red;
  }

    Có thể chỉ định rằng cụ thể những phần tử HTML sẽ bị ảnh hưởng bởi trong một lớp(<element>.<class>).

    p.center {
  text-align: center;
  color: red;
  }

    d. Universal selector

    CSS Universal Selector: chọn tất cả các phần tử HTML (*).

    *{
  text-align: center;
  color: red;
}

    e. Grouping selector

    CSS Grouping Selector: chọn các phần tử HTML để định dạng chung (,).

    h1, h2, p {
  text-align: center;
  color: red;
}

  2. CSS Combinators

    a. Descendant Selector

    CSS Descendant Selector: chọn phần tử con là con cháu của một phần tử cụ thể được chỉ định.

    div p {
  background-color: yellow;
}

    b. Child Selector

    CSS Child Selector: chỉ chọn phần tử là phần tử con của một phần tử cụ thể được chỉ định.(>)

    div > p {
  background-color: yellow;
}

    c. Adjacent Sibling Selector

    CSS CAdjacent Sibling Selector: chọn phần tử anh em liền kề phía sau một phần tử cụ thể được chỉ định.(+)

    div + p {
  background-color: yellow;
}

    d. General Sibling Selector

    CSS General Sibling Selector: chọn tất cả phần tử anh em liền kề phía sau một phần tử cụ thể được chỉ định.(~)

    div + p {
  background-color: yellow;
}

  3. CSS Pseudo-classes

    a. Anchor Pseudo-classes

    CSS Anchor Pseudo-classes: Các lớp giả này cho phép bạn xác định trạng thái hoặc vị trí của các liên kết trong tài liệu.

    /* unvisited link */
a:link {
  color: #FF0000;
}

/* visited link */
a:visited {
  color: #00FF00;
}

/* mouse over link */
a:hover {
  color: #FF00FF;
}

/* selected link */
a:active {
  color: #0000FF;
}

    Lưu ý: a:hover PHẢI đặt sau a:link và a:visited trong định nghĩa CSS để có hiệu quả! a:active PHẢI đặt sau a:hover trong định nghĩa CSS để có hiệu quả! Tên lớp giả không phân biệt chữ hoa chữ thường

    b. The :first-child Pseudo-class

    CSS The :first-child Pseudo-class: chọn phần tử con đầu tiên

    p:first-child {
  color: blue;
}

    Chọn phần tử con đầu tiên trong một phần tử cụ thể

    p i:first-child {
  color: blue;
}

    Chọn tất cả phần tử cụ thể trong phần tử con đầu tiên

    p:first-child i {
  color: blue;
}

  4. CSS Pseudo-elements

    a. The ::first-line Pseudo-element

    CSS The ::first-line Pseudo-element: được sử dụng để thêm một kiểu dáng đặc biệt cho dòng đầu tiên của một đoạn văn bản.

    p::first-line {
  color: #ff0000;
}

    b. The ::first-letter Pseudo-element

    CSS The ::first-letter Pseudo-element: được sử dụng để thêm một kiểu dáng đặc biệt cho kí tự đầu tiên của một đoạn văn bản.

    p::first-letter {
  color: #ff0000;
}

    c. The ::before Pseudo-element

    CSS The ::before Pseudo-element: để chèn nội dung nào đó trước nội dung của một phần tử.

    p::before {
  content: url(smiley.gif);
}

    d. The ::after Pseudo-element

    CSS The ::after Pseudo-element: để chèn nội dung nào đó sau nội dung của một phần tử.

    p::after {
  content: url(smiley.gif);
}

    e. The ::marker Pseudo-element

    CSS The ::marker Pseudo-element: để chọn các chỉ mục của danh sách

    p::marker {
  color: red;
}

    f. The ::selection Pseudo-element

    CSS The ::selection Pseudo-element: nhũng phần tử được chọn, tô đậm bởi người dùng

    p::selection {
  color: red;
}

  5. CSS Attribute Selectors

    a. [attribute] Selector

    CSS [attribute] Selector: chọn những phần tử chứa thuộc tính cụ thể.

    a[target] {
  background-color: yellow;
}

    b. [attribute="value"] Selector

    CSS [attribute="value"] Selector: chọn những phần tử chứa thuộc tính có giá trị cụ thể.

    a[target="_blank"] {
  background-color: yellow;
}

    c. [attribute~="value"] Selector

    CSS [attribute~="value"] Selector: chọn những phần tử chứa thuộc tính có giá trị chứa từ cụ thể (cách nhau bằng space).

    [target~="blank"] {
  background-color: yellow;
}

    d. [attribute|="value"] Selector

    CSS [attribute|="value"] Selector: chọn những phần tử chứa thuộc tính có giá trị chứa từ cụ thể (cách nhau bằng -).

    [target|="blank"] {
  background-color: yellow;
}

    e. [attribute^="value"] Selector

    CSS [attribute^="value"] Selector: chọn những phần tử chứa thuộc tính có giá trị bắt đầu bằng chuỗi kí tự cụ thể.

    [target^="blank"] {
  background-color: yellow;
}

    f. [attribute$="value"] Selector

    CSS [attribute$="value"] Selector: chọn những phần tử chứa thuộc tính có giá trị kết thúc bằng chuỗi kí tự cụ thể.

    [target$="blank"] {
  background-color: yellow;
}

    g. [attribute*="value"] Selector

    CSS [attribute*="value"] Selector: chọn những phần tử chứa thuộc tính có giá trị chứa chuỗi kí tự cụ thể.

    [target*="blank"] {
  background-color: yellow;
}

6. CSS Rule Priority

  1. Specificity

    Một giá trị định danh độ ưu tiên của các quy tắc CSS. Nó được biểu diễn bằng một bộ giá trị có 4 con số (a, b, c, d), trong đó mỗi con số đại diện cho một loại chọn phần tử hoặc quy tắc khác nhau. Các con số này xác định độ ưu tiên của một quy tắc CSS khi áp dụng kiểu dáng.

    Thứ tự ưu tiên của các loại chọn phần tử và quy tắc trong đặc thù là:

    1. Inline style: a = 1
    2. ID selectors: b = số lượng ID selectors trong quy tắc
    3. Class selectors, attribute selectors, và pseudo-classes: c = số lượng class selectors, attribute selectors, và pseudo-classes trong quy tắc
    4. Element selectors và pseudo-elements: d = số lượng element selectors và pseudo-elements trong quy tắc

    Ví dụ:

    • h1: (0,0,0,1)
    • .container: (0,0,1,0)
    • #head: (0,1,0,0)
    • body .container p: (0,0,1,2)
    • a:hover: (0,0,1,1)

    2. Đặc thù cao nhất sẽ có ưu tiên cao nhất và được áp dụng khi có xung đột giữa các quy tắc. Ví dụ: hai quy tắc với đặc thù (0,1,1,0) và (0,0,2,1) được áp dụng cho cùng một phần tử, quy tắc có đặc thù (0,0,2,1) sẽ có độ ưu tiên cao hơn và được áp dụng.

  2. Order of declaration

    Thứ tự khai báo trong CSS là cách thứ tự các quy tắc CSS được khai báo trong tệp CSS hoặc CSS của một trang HTML. Thứ tự này quyết định ưu tiên áp dụng các quy tắc CSS khi có các quy tắc xung đột hoặc ghi đè lên nhau.

    Một số quy định thứ tự khai báo trong CSS:

    • Thứ tự từ trên xuống dưới: Các quy tắc CSS được áp dụng theo thứ tự từ trên xuống dưới trong tệp CSS hoặc phần khai báo CSS.
    • Quy tắc sau ghi đè lên: Nếu hai quy tắc có cùng đặc thù, thì quy tắc được khai báo sau sẽ ghi đè lên quy tắc được khai báo trước đó.

7. CSS Attribute

Attribute(thuộc tính) bao gồm các cặp "tên thuộc tính" và "giá trị" được sử dụng để kiểm soát và định dạng các phần tử HTML trên trang web.

Mỗi thuộc tính CSS có một tên đặc trưng và có thể có một hoặc nhiều giá trị.

8. Styling Box

Là thuật ngữ được sử dụng để chỉ đến việc tạo kiểu và trang trí cho các hộp được tạo ra bởi box model.

Box Model: Là mô hình dùng để xác định cách các phần tử HTML hiển thị và tương tác trong trình duyệt. Gồm 4 phần quan trọng: content, padding, border, margin.

Box Model

Styling box bao gồm việc sử dụng các thuộc tính CSS như width, height, border, margin, padding, background, position và nhiều thuộc tính khác để kiểm soát diện mạo và vị trí của hộp trang trí.

12.Flexbox / Grid

a. Flexbox

Cấu trúc cơ bản của một phần tử flexbox với 3 flex item
  • Flex container: Chính là flexbox, nơi chứa các flex item bên trong nó.
  • Main axis: trục quyết định hướng mà các flex item sẽ hiển thị (mặc định từ trái qua phải); cross axis: trục còn lại của flexbox, vuông góc với main axis.
  • Cross start: vị trí bắt đầu của trục cross axis; cross end: vị trí kết thúc của trục cross axis.
  • Main start: vị trí bắt đầu của trục main axie; main end: vị trí kết thúc của trục ngang.
  • Flex item: các phần tử con chứa trong flexbox.
  • Main size: kích thước của flex item dựa theo hướng main axis.
  • Cross size: kích thước của flex item dựa theo hướng cross axis.

Để tạo ra một flex container, ta tiến hành khai báo:

          
            .flex-container {
              display: flex;
            }

b. Grid

Để tạo ra một grid container, ta tiến hành khai báo: 

          
            .grid-container {
              display: grid;

Tập hợp các grid item theo hàng ngang là các hàng (row)

    2. Flexbox

      Được thiết kế giúp bố cục layout theo một chiều ngang hoặc dọc.

    3. Grid Layout

      Nếu Flexbox được thiết kế cho bố cục 1 chiều, thì Grid Layout được thiết kế dạng lưới, có thể điều chỉnh bố cục cho cả 2 chiều, ngang và dọc.

    4. Positions

      Concept: là thuộc tính sử dụng để kiểm soát cách các phần tử HTML hiển thị và được định vị trên trang web. Dưới đây là một số giá trị phổ biến của position:

      1. Static: là giá trị mặc định của positon. Các phần tử sẽ được hiển thị theo trình tự tự nhiên của tài liệu HTML và sẽ không bị ảnh hưởng bởi các thuộc tính vị trí khác như: top, bottom, right, left.
      2. Relative: các phần tử sẽ được sẽ được định vị tương đối so với vị trí ban đầu của nó. Bằng cách sử dụng các thuộc tính top, bottom, left, right, phần tử có thể được di chuyển từ vị trí ban đầu một cách tương đối.
      3. Absolute: các phần tử sẽ được định vị tuyệt đối so với phần tử cha gần nhất có position khác static. Phần tử sẽ được di chuyển dựa trên các thuộc tính top, bottom, left, right. Nếu không có phần tử cha nào có position khác static, thì phần tử absolute sẽ dựa trên cửa sổ trình duyệt.
      4. Fixed: phần tử sẽ được định vị tuyệt đối so với cửa sổ trình duyệt. Nó sẽ không di chuyển khi người dùng cuộn trang. Để định rõ vị trí của phần tử fixed, ta sử dụng các thuộc tính top, bottom, left, right.

      5. Sticky: được sử dụng để tạo một phần tử có thể "dính" (sticky) trên màn hình khi người dùng cuộn trang. Nghĩa là phần tử sẽ giữ vị trí của nó trong phạm vi một thành phần cha cụ thể, cho đến khi vị trí của nó đạt đến giới hạn đã được định sẵn, sau đó nó sẽ "dính" tại vị trí đó khi cuộn tiếp tục diễn ra.

    7. Table Layout

      Trong CSS, thuộc tính display: table được sử dụng để biến một phần tử HTML thành một phần tử bảng (table). Khi một phần tử có thuộc tính display: table, nó sẽ được hiển thị và xử lý như một bảng HTML.

      Khi sử dụng display: table, các phần tử con bên trong phần tử cha sẽ được hiểu là các thành phần của bảng, bao gồm các hàng (table rows), các ô (table cells), tiêu đề (table headers), v.v.

      Example:

      First of all, tell us your name and age.

      First of all, tell us your name and age.

      Thuộc tính display: table ít được sử dụng trong CSS vì có các hạn chế và khó khăn trong việc kiểm soát và thiết kế giao diện.

      1. Hạn chế về kiểm soát giao diện: Khi sử dụng display: table, giao diện của bảng phụ thuộc vào việc tự động xử lý của trình duyệt. Việc kiểm soát chiều cao, chiều rộng và căn chỉnh các phần tử trong bảng có thể trở nên khó khăn và không linh hoạt
      2. Khó tạo kiểu tùy chỉnh: Thuộc tính display: table cung cấp một cấu trúc giao diện bảng cố định. Điều này có nghĩa là việc tạo kiểu tùy chỉnh cho các phần tử trong bảng có thể trở nên khó khăn. Một số yêu cầu thiết kế đặc biệt có thể không được đáp ứng một cách dễ dàng khi sử dụng display: table.
      3. Hiệu suất: So với các phần tử linh hoạt khác như display: flex hay grid, display: table có thể có hiệu suất kém hơn. Việc xử lý và vẽ các bảng có thể tốn nhiều tài nguyên và có thể làm chậm trang web.

    14. About CSS Stacking Contexts

    Là một khái niệm quan trọng trong CSS để xác định cách các phần tử HTML được xếp chồng và hiển thị trên trang web. Mỗi phần tử HTML có một stacking context riêng, và cách chúng xếp chồng với nhau được quy định bởi một số quy tắc.

    Một stacking context được tạo ra khi một phần tử thỏa mãn một trong những điều kiện sau:

    1. Phần tử có một thuộc tính position không phải là static (như relative, absolute hoặc fixed) và một trong những thuộc tính sau đây được áp dụng: z-index, opacity nhỏ hơn 1, transform, ...
    2. Phần tử có một thuộc tính position là fixed hoặc sticky.
    3. Phần tử là một root của trang web (ví dụ: phần tử <html>).

    Mỗi stacking context có một "độ sâu" được xác định bởi giá trị z-index của các phần tử trong nó. Phần tử với z-index lớn hơn sẽ được hiển thị trên phần tử với z-index nhỏ hơn.

    15. Animation

    CSS Animation là một công nghệ cho phép tạo ra các hiệu ứng chuyển động của các phần tử HTML. Cho phép điều chỉnh thuộc tính của một phần tử theo thời gian, tạo ra các hiệu ứng chuyển động.

    Để tạo ra một Animation trong CSS, cần thực hiện các bước sau:

    1. Định nghĩa Animation: Sử dụng @keyframes để định nghĩa các trạng thái của animation. Mỗi keyframe đại diện cho 1 trạng thái của phần tử tại 1 thời điểm cụ thể trong quá trình animation.

    2. Áp dụng Animation: Sử dụng thuộc tính animation hoặc cá thuộc tính liên quan để áp dụng animation vào phần tử.

    Cú pháp rút gọn của Animation:

    animation: name duration timing-function delay iteration-count direction fill-mode;

    Trong đó:
    • name: tên của animation được định nghĩa bằng @keyframes
    • duration: thời gian hoàn thành animation, có thể là giá trị số (ex: 1s, 500ms, ...) hoặc 'infinite' để lặp vô hạn.
    • timing-function: hàm thời gian, quyết định tốc độ chuyển động.
    • delay: độ trễ trước khi animation bắt đầu.
    • iteration-count: số lần lặp lại của animation, có thể là giá trị số hoặc infinite để lặp vô hạn.
    • direction: xác định cách animation chuyển động khi lặp lại. Nó quyết định animation sẽ chạy theo hướng nào và có đảo ngược hay không.
    • fill-mode: trạng thái của phần tử sau khi animation kết thúc.
